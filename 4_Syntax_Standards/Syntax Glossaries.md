# Glossary Syntax

```
    - `[Item]`: [Meaning]
        - [Additional Info]
        - [Additional Info]
    ```
    EXAMPLE:
        [example text]
        [example text]
    ```
    \n
```


---
### `-`: Dash
- use a dash to start a dot point below the item and definition.
- add as many dot points needed for understanding.

---
### `` ` ``: backtick/Grave Accents
- backtick are wrapped around the [Item] to create an single (inline) code block to help highlight and identify the items from the definitions.
- Located on the top Left of the Keyboard with `~` Tide symbol, left of the 1 key.

---
### `[Item]`: Abbreviation , Word or Phrase
- Replace `[Item]` with the abbreviation , word or phrase to be defined.

---
### `:`: colon symbol
- It signals that what follows will explain or elaborate on what came before.

---
### `[Meaning]`: Meaning, Definition or full-name
- define the item in basic and simple english.
- the text will automatically wrap to another line if it exceeds the viewable threshold

---
### `EXAMPLES`
- Examples are place in a multiline code block.
- with the first starting with and the last Line ending with 3 consecutive backticks.
- after the opening backticks "EXAMPLE:" is written for added clarity.
```
    ```
    EXAMPLE:
        [example text]
        [example text]
    ```
```

---
### `\n` = New Line
- while coding `\n` is used to define a new line.
- to be left out (blank) with only a blank and empty line.
- this will seperate the Items, Definitions and examples.
- with the next item starting after a blank/empty line.
